주요 해결 사항
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. ❌ 초기 문제: OTS parsing error (vhea 테이블 버전 오류)
   ├─ 원인: 폰트의 vhea 테이블 버전이 0x10001 (비표준)
   └─ 증상: 브라우저가 폰트를 파싱하지 못함

2. ✅ 해결 방법: vhea 테이블 버전 수정
   ├─ fonttools 라이브러리 사용
   ├─ vhea 테이블 버전: 0x10001 → 0x10000
   └─ WOFF2 포맷으로 변환 (907KB → 213KB)

3. ✅ 적용 방식: Base64 인코딩 + CSS 임베드
   ├─ WOFF2 파일을 base64로 인코딩
   ├─ CSS @font-face에 직접 임베드
   └─ 시스템 폰트 설치 불필요
